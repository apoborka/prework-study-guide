HTML

    Head and Metadata Tags


  • head | Both an element and a container. This is NOT displayed to users.
  • title | Pivitol for SEO and user accessibility technology.
  • style | Defines the CSS styles used.
  • base | The base URL for the app.
  • link | Connects an external resource to the HTML document.
  • meta | Defines the metadata such as the character set, page description, keywords, author, and viewport.
  • script | Embeds client-side scripts in the HTML document.

    • Body Tags


  • body | A semantic element and a container, the name of the element describes its contents and role on the webpage. This is displayed to users.
  • div | Division on the page.
  • span | Inline division on the page.
  • h1, h2, ..., h6 | Header styles. Each page should only use h1 once, and header styles should be used in sequence (h1 -> h2 -> h3, etc.).
  • p | Denotes a paragraph.
  • ul/ol | Unordered/Ordered list.
  • li | List item (nested within ul/ol)
  • img | Image element that can contain the following attributes: "src" for the imageUrl, "alt" for the imageNameAndDescription, "width", and "height".
  • br / | Line Break
  • a | Hyperlink (Anchor). Contains href="linkedUrl">linkDescription.
  • main
  • footer | Contains footer information. Use with nested p (paragraph) tag.
  • section | Container.


CSS

  • Bootcamp CSS Module

    • CSS Methodologies


  • There are 3 ways to style a webpage using CSS:
    1. Inline | Applies style rules to a specific HTML tag to style a particular page element only.
      • For quick and permenant changes, but is less flexible.
      • Future changes will require the editing of each individual style attribute used.
    2. Internal | Applies style rules for the webpage in the "head" element of the HTML file.
      • Uses CSS classes and IDs to identify elements to which styling rules will be applied.
      • Changes applied to the CSS styling rules in the head element will apply to all tags associated with that rule.
    3. External | Applies style rules in a separate file using a "link" element.
      • This is the preferred and most common CSS styling methodology.
      • This allows all styling to be contained within its own file, and edits made will affect all linked elements.
      • Requires a link element

  • [Git Bash] git touch style.css | Used to create a new file for CSS code. Keep this file in the assets directory within the main repo.
  • After CSS styling rules have been set on the style sheet, add the link element to the head element of the HTML file. (link rel="stylesheet" href="./assets/style.css")
  • Important Definititions:
    • Margin: A margin property indicates how much space we want around the outside of an element. "0" indicates that there will be no buffer zone around the element.
    • Padding: A padding property adds space around the content inside of an element. When the padding is increased, so too is the size of the element.
  • To apply a rule to all elements on the page, use "*" (asterisk). This is known as a "special wild card symbol".

    Git Flow Steps (Feature Branching)


    1. [Git Bash] git checkout main | Make "main" the active branch.
    2. [Git Bash] git checkout -b feature/featureName | Create "feature/featureName" branch and set it as the active branch.
    3. [Git Bash] git status | Confirm the status of the current branch and display modified files that have not been staged. (No changes should be present on a new feature branch.)
    4. [Git Bash] code . | Open VS Code in the active directory.
    5. [VS Code] *Add/modify code/files.*
    6. [Git Bash] git add -A | Stage all modified files.
    7. [Git Bash] git commit -m "Commit message" | Commit modified files/code to the active branch.
    8. [Git Bash] git pull origin main | Ensure the current branch is up-to-date before opening a PR. (Sync local branch with remote branch.)
    9. [Git Bash] git push origin feature/featureName | Push all changes.
    10. [Browser - Git Hub] Navigate to GitHub. On the "Code" tab, switch to the feature/featureName branch.
    11. [Browser - Git Hub] Click the "Pull requests" tab. Click "New pull request" button.
    12. [Browser - Git Hub] Set the "base" branch to "main" and the "compare" branch to the feature/featureName branch. Click "Create pull request" button.
    13. [Browser - Git Hub] Write a descriptive message. (Assign a PR reviewer.)
    14. [Browser - Git Hub] Click "Create pull request" button. Click "Merge pull request".
    15. [Git Bash] git checkout main
    16. [Git Bash] git pull origin main
    17. [Browser - Git Hub] Close any open issues on GitHub Issues tab.

    Command Modifiers


  • -A | (git add -A) Stage all modified files.
  • -m | (git commit -m "Commit message") | Add a message to a commit.

    Setup and Initialization


  • git init | Initialize an existing directory as a Git repo.
  • git clone [URL] | Retreive an entire repo from a hosted location via URL.

    • Staging, Snapshotting, and Committing Changes


  • git status | Show modified files in working directory, staged for next commit.
  • git add [file] | Add a file as it looks now to the next commit. (Stage a file for commit.)
  • git reset [file] | Unstage a file while retaining file changes in the working directory.
  • git diff | List files that have been edited but have not yet been staged.
  • git diff --staged | List of files that have been staged but not yet committed.
  • git commit -m "[Descriptive message] | Commit staged content as a new commit snapshot.

    • Branching and Merging


  • git branch | Lists branches. An asterisk (*) will appear next to the currently active branch.
  • git branch [branch-name] | Create a new branch at the current commit.
  • git checkout | Switch to another branch and check it out into your working directory.
  • git merge [branch] | Merge the specified branch's history into the current one.
  • git log | Show all commits in the current branch's history.
